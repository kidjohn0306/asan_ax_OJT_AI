작성일: 2026-07-16 (같은 날 갱신 — 문제 생성 UI 개편, 생성 작업 이력·감사 로그 실제 연동 반영)  
대상: `asan_ax_OJT_AI`  
현재 상태: 코드 수정 완료, 테스트 완료, Git 커밋·푸시 전

## 1. 한 줄 요약

기존 관리자 화면과 Google Sheets 데이터를 바로 버리지 않고 유지하면서, 문제 생성·검수·시험지·시험 배정·응시 결과를 더 안전하게 관리할 수 있도록 백엔드 저장 구조와 관리자 UI를 함께 개선했다. 이후 추가 요청으로 문제 생성 화면의 입력 방식을 버튼형으로 바꾸고, "생성 작업 이력"과 "감사 로그"를 목업이 아닌 실제 Google Sheets 데이터와 연결된 화면으로 완성했다.

### 이번 갱신에서 새로 반영된 내용

- 문제은행에서 문제를 클릭한 뒤 "← 목록으로" 버튼으로 되돌아갈 수 있다.
- 문제 생성 화면의 "평가 유형"·"대상 팀"이 드롭다운에서 버튼(토글) 방식으로 바뀌었다. 대상 팀 버튼은 Google Sheets `teams` 탭(팀 관리 화면에서 추가한 팀 포함)을 그대로 반영한다.
- 문제 생성 화면에서 근무형태·담당 공정 입력을 제거했다.
- 문제 생성 화면의 "난이도 구성"(하/중/상)을 직접 숫자로 입력할 수 있다. 총 문항 수와 합이 다르면 경고 문구가 뜬다.
- "생성 작업 이력" 화면이 더 이상 "API 미제공" 안내가 아니라, `generation_jobs` 탭을 실제로 읽어오는 화면으로 바뀌었다.
- "감사 로그" 화면도 같은 방식으로 실제 화면이 되었다. 다만 이번에는 "관리자 행동을 기록"하는 새 기능이라 대상 Google Sheets에 `audit_logs` 탭 1개가 추가되어, 운영 탭 수가 18개 → **19개**가 되었다(탭 추가는 사용자 승인 후 진행).

## 2. 이번에 지킨 기준

- 메인 대시보드는 기존 최신 UI를 유지했다.
- 기존 색상과 버튼 모양을 유지했다.
- 기존 8개 Google Sheets 탭을 삭제하지 않았다.
- 새 기능은 기존 구조와 새 구조에 함께 기록하는 방식으로 연결했다.
- 실제 데이터가 없을 때 목업 데이터를 보여주지 않는다.
- Google Sheets 오류를 로컬 데이터로 몰래 대체하지 않는다.
- 시험을 만든 뒤 원본 문제가 바뀌어도 기존 시험 내용은 변하지 않게 한다.

## 4. 화면별 변경사항

### 시험지 생성관리

주소: `/admin/exam-papers`

- `시험지 설정`: 기존 생성 기능 유지
- `시험지 보기`: 생성된 시험지 목록과 상세 정보 추가
- 상세 화면에서 문항 순서, 보기, 정답, 점수, 버전 확인 가능
- 기존 시험지를 복사해서 새 시험지로 수정하는 방식 사용
- 같은 이름의 시험지 저장 차단
- 같은 문제 구성의 시험지 저장 차단
- 점수 합계가 정확히 100점인지 검증
- HTML 저장 기능 반영

### 시험 생성관리

주소: `/admin/exams`

- 시험 시간 기본값 60분
- 합격 점수 기본값 70점
- 시험 상태: 예정·진행 중·완료
- 상태 필터와 페이지네이션 추가
- 응시자 검색 추가
- 다른 팀 사원은 검색할 수 있지만 선택할 수 없음
- 이미 등록된 정식 시험 응시자는 중복 등록 차단
- 정식 시험은 사용자당 활성 1개만 허용
- 연습 시험은 사용자당 여러 개 허용
- 시험 문제 보기 기능 추가

### 응시 현황

주소: `/admin/exams/live`

- 계획안의 운영 현황 UI 적용
- 10초마다 최신 데이터 조회
- 이전 조회가 끝나기 전에 중복 요청하지 않음
- 첫 조회 실패 시 오류 화면 표시
- 이후 갱신 실패 시 마지막 정상 데이터 유지
- 서버에 없는 진행률이나 가짜 데이터를 만들지 않음

### 문제 관리

- 문제 생성: `/admin/questions/generate/setup`
- 생성 작업 이력: `/admin/questions/generate/runs`
- 검수 대기: `/admin/questions/review`
- 문제은행: `/admin/questions/bank`
- 문제 상세: `/admin/questions/{questionId}`

문제 생성·검수·문제은행을 별도 화면과 별도 데이터 흐름으로 나누었다. 아래는 이번 갱신에서 UI가 실제로 어떻게 바뀌었는지에 대한 상세 설명이다.

#### 4-1. 문제 생성 화면 (`/admin/questions/generate/setup`)

**출제 대상 카드**

| 항목 | 이전 | 현재 |
|---|---|---|
| 평가 유형 | `<select>` 드롭다운 (기초고사 / 업무능력평가) | 버튼 2개 중 하나를 클릭해 선택하는 토글 그룹. 선택된 버튼은 파란 테두리 + 옅은 배경으로 강조 |
| 대상 팀 | `<select>` 드롭다운 | 팀 개수만큼 버튼이 가로로 나열되는 토글 그룹 (예: `1팀` `2팀` `3팀` `안전팀`) |
| 근무형태 | `<select>` (주간 / 4조3교대 / 3조2교대) | **삭제** — 실제로 백엔드에서 구조화된 값으로 쓰이지 않고 텍스트로만 끼워 넣던 값이라 제거 |
| 담당 공정 | `<select>` (공통 / Reflow) | **삭제** (사유 동일) |

대상 팀 버튼 목록은 화면이 열릴 때 `GET /api/admin/teams`를 호출해 만들어진다. 즉 "시스템 관리 → 팀 관리" 화면(Google Sheets `teams` 탭)에서 새 팀을 추가하면, 별도 배포 없이 문제 생성 화면의 버튼 목록에도 그 팀이 바로 나타난다. 이번에 "안전팀(S2)"을 실제로 선택해 AI 문제 생성까지 성공하는 것을 확인했다.

**문항 구성 카드**

- 공통 / 팀 / 환경안전 / 일반상식 문항 수는 기존과 동일하게 숫자 입력.
- **새로 추가된 부분**: 그 아래 "난이도 구성"에 하 / 중 / 상 숫자 입력 3칸이 생겼다. 기존에는 총 문항 수의 32%/48%/나머지로 자동 계산해 배지(읽기 전용 텍스트)로만 보여줬는데, 이제 관리자가 직접 숫자를 바꿀 수 있다.
- 문항 수를 바꾸면 난이도 구성 값도 비율에 맞춰 자동으로 다시 계산된다. 단, 관리자가 난이도 값을 한 번이라도 직접 수정하면 그 뒤로는 자동 재계산을 멈추고 관리자가 입력한 값을 그대로 유지한다(총 문항 수를 다시 바꾸기 전까지).
- 하/중/상 합계가 총 문항 수와 다르면 "난이도 합계 N문항이 총 문항 수 M문항과 다릅니다." 경고 문구가 카드 안에 표시된다. 저장을 막지는 않고, 안내만 한다.

**자료 선택 / 생성 정책 카드**: 변경 없음.

#### 4-2. 문제은행 화면 (`/admin/questions/bank`, 상세 `/admin/questions/{id}`)

- 기존 문제: 문제은행 목록에서 문제 하나를 클릭하면 상세 화면(`/admin/questions/{id}`)으로 이동하는데, 이때 왼쪽 목록이 클릭한 문제 1건으로 줄어들어 버렸고, 전체 목록으로 돌아갈 방법(뒤로가기 버튼)이 화면 안에 없었다.
- 수정 내용: 상세 화면 상단(제목 옆 액션 영역)에 **"← 목록으로"** 버튼을 추가했다. 클릭하면 `/admin/questions/bank`로 이동해 필터·페이지가 초기화된 전체 목록이 다시 보인다.
- 목록이 1건으로 줄어드는 동작 자체는 그대로 유지했다(잘못된 문제 코드로 접근했을 때 "문제가 없습니다."를 정확히 보여주는 기존 테스트와 충돌하지 않도록, 목록 필터링 로직은 건드리지 않고 되돌아가는 길만 추가했다).

#### 4-3. 생성 작업 이력 (`/admin/questions/generate/runs`)

- 기존: "현재 API에서 제공되지 않는 기능입니다" + "돌아가기" 링크만 있는 빈 안내 화면.
- 변경: 실제 화면으로 교체. 백엔드에 `GET /api/admin/generation-jobs`를 새로 만들어 `generation_jobs` Google Sheets 탭(AI 문제 생성을 실행할 때마다 이미 기록되고 있던 데이터)을 그대로 읽어온다.
- 화면 구성: 작업 ID / 요청자 / 상태(배지) / 요청 수 / 완료 / 실패 / 시작 시각 열을 가진 표. 최신 작업이 위로 오도록 정렬.
- 데이터가 아직 없으면 "아직 생성 작업이 없습니다."라는 정직한 빈 상태 문구를 보여주고, 가짜 행을 만들지 않는다. (참고: 이번 검증 과정에서 만든 테스트 작업은 검증 후 모두 삭제했기 때문에, 현재는 빈 상태가 정상이다.)
- 저장소 자체가 꺼져 있는 경우(`OJT_USE_CANDIDATE_TAB=false`)에는 "생성 작업 이력 저장소가 비활성화되어 있습니다."라고 표시해, 데이터가 없는 것과 기능이 꺼진 것을 구분해서 보여준다.

#### 4-4. 감사 로그 (`/admin/system/audit-logs`)

- 기존: 생성 작업 이력과 마찬가지로 "API 미제공" 안내 화면뿐이었다.
- 변경: 관리자의 실제 행동(현재는 **문제 승인 / 문제 반려**)을 기록하는 신규 기능을 만들고, 그 기록을 보여주는 화면으로 교체했다.
- 이를 위해 대상 Google Sheets에 `audit_logs` 탭을 새로 추가했다(사용자 승인 후 진행, 18탭 → 19탭). 문제 승인·반려 시점에 "누가(사번) / 언제 / 무엇을(승인/반려) / 어떤 대상에게(문제 ID) / 어떤 사유로" 한 줄이 자동으로 기록된다.
- 화면 구성: 시각 / 수행자 / 행동(문제 승인·문제 반려로 한글 라벨링) / 대상(`question:Q-1` 형태) / 사유 열을 가진 표. 최신 기록이 위로.
- 실제 승인 1건을 테스트로 실행해 화면에 `admin001 / 문제 승인 / question:CODEX_SMOKE_AUDIT` 행이 정확히 나타나는 것을 확인한 뒤, 테스트 데이터는 삭제했다.
- 감사 로그 기록 실패(예: 일시적인 Sheets 오류)가 있어도 승인·반려 자체는 실패하지 않도록 만들었다 — 감사 로그는 "최선 노력" 방식이며 핵심 업무를 막지 않는다.
- 현재는 문제 승인·반려만 기록한다. 팀 생성, 시험 배정 등 다른 관리자 행동은 아직 감사 대상이 아니며, 필요해지면 같은 방식으로 확장할 수 있다.

## 5. 백엔드와 데이터 구조 변경

### 문제 생성·검수

문제가 만들어진 뒤 다음 단계를 각각 기록한다.

1. 생성 작업
2. 후보 문제
3. 자동 품질 검사 결과
4. 관리자 승인 또는 반려
5. 문제 변경 이력
6. 기존 문제은행 반영

자동 검사는 V01~V07 일곱 단계로 관리한다. 누가 언제 승인·반려했는지 추적할 수 있다.

### 시험지와 시험

- 승인된 문제만 시험지에 포함
- 시험지 최대 100문항
- 각 문항 점수는 양수
- 전체 점수는 정확히 100점
- 시험 생성 시 문제·보기·정답·점수를 복사해 고정
- 나중에 문제은행 원본을 수정해도 이미 만든 시험은 바뀌지 않음

### 응시 결과

문항별 답안을 `result_answers`에 저장하고, 전체 응시 상태를 `exam_attempts`에 저장한다. 기존 `results` 탭에도 함께 저장하여 기존 화면과 호환한다.

- 답하지 않은 문항도 0점으로 저장
- 같은 제출 요청을 다시 보내도 결과가 중복되지 않음
- 같은 요청 ID인데 내용이 다르면 409 충돌 반환

### 추가된 탭 1개 (감사 로그 기능)

`audit_logs` — "감사 로그" 화면을 목업이 아닌 실제 기능으로 만들면서 새로 추가했다. 사용자에게 "18탭 제약을 벗어나 19번째 탭을 추가해도 되는지" 별도로 확인받은 뒤 진행했다. 현재 운영 시트는 **총 19개 탭**이다.

```text
구조 문제: 0
```

이후 `audit_logs` 탭을 추가하면서 총 탭 수는 19개가 되었다. `migrate_schema_lean.py`는 최초 설계대로 18개 탭만 관리 대상으로 삼기 때문에, `audit_logs` 탭 추가는 이 스크립트가 아니라 별도의 단발성 작업으로 진행했고, 기존 18개 탭 구조에는 영향이 없음을 확인했다.

## 7. 중요한 저장 원칙

### 이중 기록

새 기능을 저장할 때 기존 탭과 새 탭에 모두 기록한다. 기존 기능을 갑자기 깨뜨리지 않고 새 기능을 단계적으로 도입하기 위한 방식이다.

### 멱등성

같은 요청이 재전송되어도 시험·배정·결과가 중복 생성되지 않는다.

### 엄격한 Sheets 모드

현재 `OJT_STRICT_SHEETS_STORAGE=true`다. Google Sheets 저장에 실패하면 성공처럼 보이지 않고 오류를 반환한다.

## 11. 아직 구현되지 않은 범위

다음 화면은 경로와 정직한 unavailable 상태까지 준비되어 있지만, 완전한 기능은 후속 작업이다.

- 생성 작업 "상세"(개별 작업 클릭 시 나오는 세부 화면, `/admin/questions/generate/runs/{id}`) — 목록은 실제 데이터로 연동됐지만 상세 페이지는 아직 미구현
- 문제 변경 이력 전용 화면 (`/admin/questions/{id}/history`)
- 결과 상세 전용 화면 (`/admin/results/{id}`)
- 자료 정규화 탭(`materials`, `material_slides`, `knowledge_units`)
- 응시 현황의 강제 종료·시간 연장 기능
- 감사 로그의 대상 확대 — 현재는 문제 승인·반려만 기록. 팀 생성·수정, 시험 배정·취소, 사용자 승인 등은 아직 감사 대상이 아니다.

서버가 제공하지 않는 데이터는 프론트에서 임의로 만들어 보여주지 않도록 했다. (생성 작업 이력·감사 로그는 이번 갱신으로 이 목록에서 제외되어 실제 기능이 되었다.)

## 13. 팀원이 이어서 작업할 때 주의할 점

- 현재 변경은 아직 Git 커밋·푸시 전이다.
- `git reset --hard`, `git clean -fd`로 작업 내용을 삭제하지 않는다.
- `backend/.env`와 `backend/credentials/service_account.json`은 커밋하지 않는다.
- 새 기능은 기존 탭과 새 탭의 이중 기록 정책을 확인한 뒤 수정한다.
- 반복문 안에서 Google Sheets API를 문항별로 호출하지 않는다.
- 시트 마이그레이션은 항상 Dry-run부터 실행한다.
- 다른 팀원이 원격 브랜치를 업데이트했다면 현재 변경을 먼저 보존하고 전체 테스트 후 병합한다.
