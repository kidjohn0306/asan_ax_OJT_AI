# 시험 생성·관리 및 응시 현황 목업 반영 계획

## 목표

승인된 시험 관리 목업의 핵심 흐름을 실제 관리자 앱의 `시험 생성·관리`와 `응시 현황`에 반영한다. 대시보드는 현재 `main` 구현을 그대로 유지한다.

## 확정 범위

- 시험 생성·관리
  - 월간 달력 보기와 기존 목록 보기 전환
  - 날짜 선택 후 시험 생성 흐름 진입
  - 일정 미정 시험 별도 표시
  - 팀·검색 조건으로 시험지를 비교하는 선택 레이어
  - 선택한 시험지의 팀을 읽기 전용으로 표시
  - 기존 시험 편집, 문제 보기, 삭제, 응시자 배정 기능 유지
- 응시 현황
  - 오늘, 이번 주, 전체 빠른 기간 필터
  - 월간 달력 날짜 필터
  - 기존 시험 상태·팀 필터와 URL 상태 유지
  - 10초 폴링, 오류 표시, 상세 화면 진입 유지

## 제외 범위

- 대시보드 UI와 대시보드 시험 일정 컴포넌트 변경
- 시험지 팀 또는 시험 회차 팀을 임의로 변경하는 기능
- 백엔드 일정 충돌 차단, 시간대 정규화, 수명주기 모델 개편
- 원격 저장소 푸시

## 구현 순서와 검증

1. 현재 화면과 테스트 계약을 고정한다.
   - 검증은 기존 시험 관리·응시 현황 테스트의 기대 동작을 확인한다.
2. 재사용 가능한 월간 일정 달력을 추가한다.
   - 검증은 날짜 선택, 월 이동, 일정 상태 표시를 컴포넌트 테스트로 확인한다.
3. 시험 생성·관리 화면에 달력과 시험지 선택 레이어를 연결한다.
   - 검증은 선택한 시험지 ID와 기존 생성 API 요청 본문이 일치하는지 확인한다.
4. 응시 현황 화면에 날짜 범위 필터를 연결한다.
   - 검증은 URL 쿼리, 상태·팀 필터, 폴링이 함께 유지되는지 확인한다.
5. 전체 프런트엔드 테스트와 빌드를 실행한다.
   - 검증은 `npm test`와 `npm run build`가 모두 성공해야 한다.
6. 대시보드 무변경과 로컬 커밋 상태를 확인한다.
   - 검증은 대시보드 코드 구간에 변경이 없고 원격 푸시가 발생하지 않았는지 확인한다.

## 완료 기준

- 시험 생성·관리에서 달력과 목록을 전환할 수 있다.
- 빈 날짜를 선택해 해당 날짜가 입력된 시험 생성 흐름을 시작할 수 있다.
- 시험지는 단순 선택 상자 대신 상세 선택 레이어에서 고른다.
- 시험지 선택 후 대상 팀은 변경 불가능한 값으로 표시된다.
- 응시 현황은 오늘, 이번 주, 전체, 특정 날짜로 조회할 수 있다.
- 기존 편집·삭제·문제 보기·응시자 배정과 10초 폴링이 회귀하지 않는다.
- 대시보드에는 변경이 없다.
- 로컬 검증과 로컬 커밋까지만 수행하고 푸시하지 않는다.
