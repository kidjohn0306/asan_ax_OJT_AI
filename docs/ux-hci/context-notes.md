# UX/HCI 컨텍스트 노트

## 2026-07-23 시험 관리·달력 고도화

### 사용자 요구

- 시험 일정 기능이 현재 조회 중심이라 운영 기능으로 고도화할 필요가 있다.
- 시험 생성·관리 화면에서도 달력을 활용하고 싶다.
- 팀별 시험지를 단순 선택 상자가 아닌 상세 팝업에서 고르고 싶다.
- 응시 현황에서도 날짜를 기준으로 시험을 탐색하고 싶다.
- 이번 단계에서는 분석만 수행하고 코딩하지 않는다.

### 현재 구조에서 확인한 사실

- 시험지를 만들 때 `team_code`가 필수로 저장된다.
- 시험 회차는 `exam_set_id`로 원본 시험지를 선택하고 원본의 팀, 문항, 배점, 시험 유형을 상속한다.
- 시험 생성 화면의 대상 팀은 수정 입력이 아니라 선택한 시험지에서 파생된 읽기 전용 값이다.
- 응시자 배정 시 응시자 팀과 시험 팀이 다르면 서버가 요청을 거부한다.
- 대시보드와 시험 관리 화면은 일정이 없거나 해석 불가능한 시험도 `예정`으로 처리한다.
- 응시 현황 화면은 같은 경우를 `일정 미정`으로 처리해 화면 간 상태 규칙이 다르다.
- 일정 입력은 시간대가 없는 `datetime-local` 문자열이며 서버는 형식과 일정 충돌을 검증하지 않고 저장한다.
- 대시보드 캘린더는 `Admin.jsx` 내부 전용 구현이라 시험 관리와 응시 현황에서 재사용할 수 없다.
- 응시 현황은 10초마다 시험 목록 한 번과 시험별 결과 N번을 요청한다.
- V2 시트 스키마에는 `scheduled_start_at`, `scheduled_end_at`, `lifecycle_status`, `live_status`가 있지만 현재 화면은 주로 `exam_datetime`과 `duration_min`으로 상태를 추정한다.

### 권장 결정

- 시험지 팀은 원본의 출제 범위이므로 시험 회차에서 임의 변경하지 않는다.
- 시험지 선택 팝업에서 팀을 먼저 필터링하고, 선택 후 대상 팀을 명확히 고정 표시한다.
- 다른 팀에서 같은 구성을 쓰려면 시험지 수정본을 만들거나 향후 `공통 시험지` 범위를 별도 모델로 도입한다.
- 대시보드 캘린더는 요약과 진입점, 시험 관리 캘린더는 생성·수정, 응시 현황 캘린더는 날짜 필터 역할을 맡는다.
- 일정 상태는 서버가 시간대와 수명주기를 기준으로 계산한 값을 단일 기준으로 제공한다.
- 1차 구현은 기존 저장 구조와 호환되는 조회·선택 UX 개선으로 제한하고, 일정 충돌과 수명주기 정규화는 후속 백엔드 단계로 분리한다.

### 보류 사항

- 공통 시험지를 여러 팀에 직접 배정할 실제 업무 요구가 있는지 확인이 필요하다.
- 같은 팀의 시험 시간이 겹칠 때 차단할지 경고만 할지 제품 정책 확정이 필요하다.
- 시험 시작 후 일정 변경을 완전 금지할지 운영자 승인 흐름을 둘지 결정이 필요하다.
- 달력 1차 범위를 월 보기만으로 할지 주 보기까지 포함할지 결정이 필요하다.

## 2026-07-23 시험 관리·달력 목업

### 작업 경계

- 사용자는 분석안이 적용된 목업을 요청했다.
- 사용자는 별도 요청 전까지 원격 푸시를 절대 하지 말라고 명시했다.
- 목업은 `mockup/index.html`의 독립 실행형 프런트엔드 파일로 만든다.
- 기존 React, FastAPI, API 계약, 저장 데이터는 수정하지 않는다.

### 목업에서 채택한 기본값

- 시험지 팀은 시험 회차에서 변경하지 않고 읽기 전용으로 상속한다.
- 달력 1차 범위는 월 보기와 날짜별 일정 패널로 제한한다.
- 같은 팀의 정식 시험 시간 중복은 차단 경고로 표현한다.
- 대시보드는 일정 편집 화면이 아니라 운영 경고와 시험 관리 진입점으로 표현한다.
- 응시 현황은 오늘을 기본값으로 하고 달력은 조회 날짜를 바꾸는 필터로 사용한다.
- 시험지 선택은 팀 필터, 버전, 문항 수, 사용 회차, 난이도·카테고리 구성을 비교하는 팝업으로 표현한다.

### 검증 계획

- 단순 정적 캡처가 아니라 화면 전환, 날짜 선택, 시험지 선택, 보기 전환을 직접 조작한다.
- 인앱 브라우저에서 데스크톱 화면과 좁은 화면의 레이아웃을 확인한다.
- 최종 확인 후 로컬 커밋까지만 수행하고 원격 푸시는 하지 않는다.

### 검증 결과

- `http://127.0.0.1:4317/`에서 HTTP 200 응답을 확인했다.
- 대시보드에서 운영 경고, KPI, 월간 일정이 표시되는 것을 확인했다.
- 시험 관리에서 빈 날짜를 선택하면 해당 날짜가 시험 생성 패널에 반영되는 것을 확인했다.
- 시험지 선택 팝업의 팀 필터, 시험지 행 선택, 버전·문항 구성 미리보기를 확인했다.
- 2팀 시험지를 선택했을 때 대상 팀이 `잠금 · 2팀 시험지에서 상속됨`으로 표시되는 것을 확인했다.
- 일정 단계에서 선택 날짜와 충돌 없음 안내가 일치하는 것을 확인했다.
- 응시자 확인 단계에서 선택한 시험지와 같은 팀의 승인 사용자만 표시되도록 구현했다.
- 시험 관리의 달력 보기와 목록 보기가 전환되는 것을 확인했다.
- 응시 현황의 오늘·이번 주 전환과 집계 변경을 확인했다.
- 브라우저 콘솔 오류가 없음을 확인했다.
- 1024px 너비와 390px 너비에서 문서 가로 넘침이 없음을 확인했다.
- 390px 너비에서 시험지 팝업이 전체 화면으로 표시되고 내부 콘텐츠를 세로로 탐색할 수 있음을 확인했다.
- 기존 애플리케이션 소스는 수정하지 않았다.
- 원격 푸시는 수행하지 않았다.
