# 시험 관리·달력 고도화 분석 계획

## 2026-07-23

### 목적

시험지 선택, 시험 회차 생성, 일정 조회, 응시 현황이 하나의 운영 흐름으로 이어지도록 현재 구현과 데이터 계약을 분석한다.
이번 작업은 요구사항과 설계 방향을 정리하는 문서 작업이며 애플리케이션 코드는 변경하지 않는다.

### 분석 범위

- 대시보드의 시험 일정 캘린더.
- 시험 생성·관리 화면의 시험지 선택과 일정 입력.
- 응시 현황 화면의 날짜 탐색과 필터.
- 시험지, 시험 회차, 팀, 응시자 배정 사이의 데이터 규칙.
- 일정 상태, 시간대, 충돌 검증, 수정 제한에 필요한 서버 계약.

### 분석 순서

1. 관련 프런트엔드 화면과 API 호출을 추적한다.
2. 시험지와 시험 회차를 생성하는 백엔드 계약을 확인한다.
3. 현재 달력과 상태 판정 방식의 불일치를 찾는다.
4. 팀별 시험지 선택 로직의 대안을 비교한다.
5. 권장 사용자 흐름과 단계별 구현 범위를 정의한다.
6. 구현 전에 합의할 결정사항과 인수 기준을 남긴다.

### 성공 기준

- 시험지와 시험 회차의 역할이 명확히 구분된다.
- 팀이 어느 단계에서 결정되고 변경 가능한지 일관된 규칙이 정해진다.
- 대시보드, 시험 생성·관리, 응시 현황에서 달력의 역할이 중복 없이 정의된다.
- 일정 데이터의 서버 기준 상태와 충돌 검증 방향이 제시된다.
- 1차 구현 범위와 후속 구조 개선 범위가 분리된다.
